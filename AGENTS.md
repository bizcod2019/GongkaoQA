# 公务员面试知识问答系统

## 项目概览

这是一个基于纯静态HTML/CSS/JavaScript的问答系统，旨在帮助用户记忆公务员面试结构化面试题型模板的核心知识点。

### 技术栈
- **前端**: 纯HTML5 + CSS3 + JavaScript (原生)
- **部署**: 静态文件服务器，可部署到GitHub Pages
- **特点**: 无需后端，纯前端实现

### 核心功能
1. **100道精选题目** - 覆盖五大题型：
   - 人际沟通题
   - 应急应变题
   - 组织管理题
   - 综合分析题
   - 自我认知与岗位匹配题

2. **自定义题目数量** - 用户可选择：
   - 10题（快速测试）
   - 20题（标准模式）
   - 30题（默认推荐）
   - 50题（深入学习）
   - 100题（完整挑战）

3. **即时反馈系统**
   - ✅ 正确答案：显示庆祝动画（彩带、烟花）
   - ❌ 错误答案：显示悲伤表情 + 正确答案 + 详细解析

4. **学习跟踪**
   - 实时显示答题进度
   - 统计正确/错误数量
   - 计算正确率
   - 错题回顾功能

## 开发说明

### 如何运行
项目已通过 `coze init` 初始化，使用原生静态服务器运行：
- 开发环境：`coze dev` (端口 5000)
- 访问地址：`http://localhost:5000`

### 如何修改题目
题目数据存储在 `questions-data.js` 文件中：
```javascript
const allQuestions = [
  {
    "id": 1,
    "question": "题目内容",
    "options": ["A. 选项1", "B. 选项2", "C. 选项3", "D. 选项4"],
    "answer": "B",
    "explanation": "解析内容"
  },
  // ... 更多题目（共100道）
];
```

### 如何修改题目数量选项
在 `index.html` 中修改题目数量选择器：
```html
<div class="count-options">
  <button class="count-option" data-count="10">10 题</button>
  <button class="count-option active" data-count="30">30 题</button>
  <!-- 可以添加或修改这里的选项 -->
</div>
```

### 如何修改样式
- 主样式文件：`styles/main.css`
- 主要颜色配置：
  - 主色调：`#667eea` → `#764ba2` (渐变紫色)
  - 正确颜色：`#00b894` (绿色)
  - 错误颜色：`#d63031` (红色)

### 如何添加新功能
由于是纯静态项目，所有功能都在 `index.html` 中实现：
- 修改HTML结构：编辑 `index.html` 的 `<body>` 部分
- 修改JavaScript逻辑：编辑 `index.html` 的 `<script>` 部分
- 修改样式：编辑 `styles/main.css`

## 部署说明

### GitHub Pages 部署步骤
1. 将整个项目推送到GitHub仓库
2. 进入仓库的 Settings → Pages
3. 选择分支（通常是 main 或 master）
4. 选择根目录 `/` 作为源
5. 点击 Save，等待部署完成

### 注意事项
- 项目完全静态，无需任何构建步骤
- 所有资源都是相对路径，部署后可直接访问
- 支持移动端响应式设计

## 常见问题

### Q: 如何修改题目数量？
A: 在开始页面可以选择10、20、30、50、100道题目。修改 `index.html` 中的 `.count-option` 按钮可自定义选项。

### Q: 题目是如何随机抽取的？
A: 系统使用 Fisher-Yates 洗牌算法从100道题库中随机抽取用户选择的数量的题目，确保每次答题题目不重复且随机。

### Q: 如何添加更多题目？
A: 编辑 `questions-data.js` 文件，在 `allQuestions` 数组中添加新的题目对象即可。

### Q: 如何修改庆祝动画效果？
A: 编辑 `showCelebration()` 函数和对应的CSS动画关键帧。

### Q: 如何添加新的题型？
A: 在 `questions` 数组中添加新题目，并在开始页面的介绍中更新题型列表。

### Q: 为什么题目数据要硬编码？
A: 为了保持纯静态特性，无需后端API，可直接部署到GitHub Pages等静态托管平台。
